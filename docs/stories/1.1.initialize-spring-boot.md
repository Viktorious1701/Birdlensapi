# Story 1.1: Initialize containerized Spring Boot 3 application

## Status
Done

## Story
**As a** Backend Developer,
**I want** to initialize a containerized Spring Boot 3 application with a unified docker-compose.yml
**so that** I can reliably run PostgreSQL (PostGIS), Redis, RabbitMQ, and MinIO locally without installing dependencies on my host machine.

## Acceptance Criteria
1. A Spring Boot 3 application (birdlensAPI) is initialized with Web, Data JPA, Security, Validation, and Actuator dependencies.
2. A docker-compose.yml file at the repository root successfully spins up isolated containers for PostgreSQL (with PostGIS extension), Redis, RabbitMQ, and MinIO.
3. The Spring Boot application successfully connects to the PostgreSQL database on startup.
4. A basic /api/v1/health endpoint (unsecured) returns a 200 OK status, verifying the application context loads.

  - [x] Verify `build.gradle` has Web and Data JPA (already present).

## Dev Notes
- The repository is already initialized with Spring Boot 3.4.x (Java 17). Ensure `build.gradle` is updated with the remaining dependencies.
- **Architecture Reference**:
  - PostgreSQL credentials: Use `DB_USERNAME=birdlens` and `DB_PASSWORD=secret` for local dev.
  - Make sure to ignore `.env` files in `.gitignore` and create an `.env.example` as required by the architecture.
  - Keep the security configuration minimal for now—just enough to allow the health endpoint to pass without a 401 Unauthorized error.
