# Story 1.1: Initialize containerized Spring Boot 3 application

## Status
Draft

## Story
**As a** Backend Developer,
**I want** to initialize a containerized Spring Boot 3 application with a unified docker-compose.yml
**so that** I can reliably run PostgreSQL (PostGIS), Redis, RabbitMQ, and MinIO locally without installing dependencies on my host machine.

## Acceptance Criteria
1. A Spring Boot 3 application (birdlensAPI) is initialized with Web, Data JPA, Security, Validation, and Actuator dependencies.
2. A docker-compose.yml file at the repository root successfully spins up isolated containers for PostgreSQL (with PostGIS extension), Redis, RabbitMQ, and MinIO.
3. The Spring Boot application successfully connects to the PostgreSQL database on startup.
4. A basic /api/v1/health endpoint (unsecured) returns a 200 OK status, verifying the application context loads.

## Tasks / Subtasks
- [ ] Task 1: Update Spring Boot dependencies (AC: 1)
  - [ ] Verify `build.gradle` has Web and Data JPA (already present).
  - [ ] Add `spring-boot-starter-security`, `spring-boot-starter-validation`, and `spring-boot-starter-actuator` to `build.gradle`.
  - [ ] Add PostgreSQL JDBC driver dependency.
- [ ] Task 2: Create local infrastructure via Docker Compose (AC: 2)
  - [ ] Create `docker-compose.yml` in the root directory.
  - [ ] Add `postgres` service using image `postgis/postgis:16-3.4` (Port 5432).
  - [ ] Add `redis` service using image `redis:7-alpine` (Port 6379).
  - [ ] Add `rabbitmq` service using image `rabbitmq:3.13-management` (Ports 5672 / 15672).
  - [ ] Add `minio` service using image `minio/minio:latest` (Ports 9000 / 9001).
- [ ] Task 3: Configure Database Connection (AC: 3)
  - [ ] Update `src/main/resources/application.properties` (or rename to `.yml`) with database connection strings pointing to the Docker Compose PostgreSQL instance.
  - [ ] Set up environment variables locally for database credentials.
- [ ] Task 4: Create Health Check Endpoint (AC: 4)
  - [ ] Create a `HealthController` mapped to `/api/v1/health`.
  - [ ] Configure Spring Security to permit all unauthorized requests to `/api/v1/health`.

## Dev Notes
- The repository is already initialized with Spring Boot 3.4.x (Java 17). Ensure `build.gradle` is updated with the remaining dependencies.
- **Architecture Reference**:
  - PostgreSQL credentials: Use `DB_USERNAME=birdlens` and `DB_PASSWORD=secret` for local dev.
  - Make sure to ignore `.env` files in `.gitignore` and create an `.env.example` as required by the architecture.
  - Keep the security configuration minimal for now—just enough to allow the health endpoint to pass without a 401 Unauthorized error.

### Testing
- Run `docker-compose up -d` and verify all 4 infrastructure containers are healthy.
- Run the Spring Boot application locally and verify it connects to the database without crashing.
- Execute a `GET` request to `http://localhost:8080/api/v1/health` and expect a `200 OK`.
