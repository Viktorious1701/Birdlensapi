# Story 4.4: Asynchronous Image Processing Worker

## Status
Done

## Story
**As the** System,
**I want** a dedicated background worker to consume PostCreatedEvent messages and process the associated images,
**so that** users have fast-loading thumbnails and optimized images for their feed.

## Acceptance Criteria
1. A `@RabbitListener` is implemented to consume messages from the `image-processing-queue`.
2. The consumer is only active when the Spring profile `worker` is enabled.
3. For each `postId` in the event, the worker fetches the associated `post_media` records.
4. The worker downloads the original image from MinIO/S3.
5. The worker generates a 300x300 thumbnail and a web-compressed (quality ~70%) version of each image using a library like `Thumbnailator`.
6. The worker uploads the processed versions back to MinIO/S3 and updates the `post_media` record with the new URLs and sets status to `COMPLETED`.
7. Failed processing attempts are retried 3 times with backoff before being sent to a Dead-Letter Queue (DLQ).
8. An integration test verifies the full async flow: publish event -> worker processes -> DB record status is COMPLETED.
