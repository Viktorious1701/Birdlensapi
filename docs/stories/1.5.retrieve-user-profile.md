## Story
**As an** Authenticated Mobile User,
**I want** to retrieve my profile information securely,
**so that** the app can display my avatar, subscription status, and username.

## Acceptance Criteria
1. A `GET /api/v1/users/me` endpoint is created.
2. Spring Security intercepts the request, validates the provided JWT in the `Authorization: Bearer <token>` header, and extracts the user ID.
3. The endpoint returns a `200 OK` with the authenticated user's full profile data (id, email, username, first_name, last_name, avatar_url, subscription).
4. Requests missing a valid token, or carrying an expired token, return a `401 Unauthorized`.
5. A Spring Security MockMvc test (or WebTestClient integration test) verifies that an unauthenticated request is blocked, and an authenticated request with a mocked JWT succeeds.

## Dev Notes
- **Security Context**: The `JwtAuthFilter` already extracts the user's email and populates the `SecurityContextHolder`. You can access the authenticated user via `@AuthenticationPrincipal UserDetails userDetails` in the controller method signature.
- **DTOs**: The `UserResponse` record currently exists but lacks `subscriptionId` and `subscriptionExpiresAt`. You must add these fields to map the full profile data to satisfy AC 3.
- **Testing**: Use `AbstractIntegrationTest` with `WebTestClient` as established in `AuthIntegrationTest`. Do not mock the security context; acquire a real JWT by logging in a test user first.

### Testing
- Test file location: `src/test/java/com/example/birdlensapi/integration/UserIntegrationTest.java`
