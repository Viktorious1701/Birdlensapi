# Story 4.6: Social Interactions (Comments and Likes)

## Status
Done

## Story
**As a** Mobile Client,
**I want** to add comments and toggle likes on posts,
**so that** I can interact with the community.

## Acceptance Criteria
1. An endpoint `POST /api/v1/posts/{postId}/reactions` is implemented to toggle a "Like" reaction.
2. The reaction logic handles toggling: if a user "likes" a post they have already liked, the existing reaction is deleted (Unlike). If not, a new reaction is created.
3. An endpoint `POST /api/v1/posts/{postId}/comments` allows users to add a text comment to a post.
4. An endpoint `GET /api/v1/posts/{postId}/comments` returns a paginated list of comments for a specific post.
5. All interaction endpoints are secured with JWT authentication.
6. The `PostFeedResponse` (from Story 4.5) is updated to include a `likeCount` and `commentCount` (or a `userHasLiked` boolean).
7. An integration test verifies the toggle logic for likes and the pagination of comments.

## Dev Notes
- **Hibernate `@Formula` usage**: Applied to `likeCount` and `commentCount` inside the `Post` entity. This executes native aggregate subqueries seamlessly during entity hydrations preventing N+1 loop polling across social graph relations.
- **Cache Eviction strategy**: At present, standard Redis TTL expiry flushes the feed natively. Future optimization targets (`@CacheEvict`) or localized state management on Mobile apps handles instantaneous feedback rendering.

### Testing
