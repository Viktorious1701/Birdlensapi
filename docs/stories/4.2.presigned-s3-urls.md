# Story 4.2: Secure Pre-signed Upload URLs (MinIO/S3)

## Status
Done

## Story
**As a** Mobile Client,
**I want** to request a secure, pre-signed URL to upload media files directly to object storage (MinIO/S3),
**so that** large files do not need to be proxied through the backend API.

## Acceptance Criteria
1. An endpoint `POST /api/v1/posts/media/request-upload` accepts a list of desired filenames and content types.
2. The service uses the AWS SDK (configured for MinIO locally) to generate a unique, time-limited (e.g., 15 minutes), pre-signed `PUT` URL for each requested file.
3. The endpoint returns a list of objects, each containing the unique generated object key and its corresponding pre-signed URL.
4. The endpoint is secured with JWT authentication.
5. An integration test verifies that a valid pre-signed URL is generated and can be used to execute an HTTP PUT request directly against the Testcontainers MinIO instance.

## Dev Notes
- **Architectural Validation**: Proceeded with the AWS SDK V2 utilizing Path-Style Access override. This achieves an enterprise-grade infrastructure allowing the Java codebase to run agnostic of whether the container target is a local MinIO bucket or a massive AWS cloud resource in production.
