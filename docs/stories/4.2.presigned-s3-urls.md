# Story 4.2: Secure Pre-signed Upload URLs (MinIO/S3)

## Status
Draft

## Story
**As a** Mobile Client,
**I want** to request a secure, pre-signed URL to upload media files directly to object storage (MinIO/S3),
**so that** large files do not need to be proxied through the backend API.

## Acceptance Criteria
1. An endpoint `POST /api/v1/posts/media/request-upload` accepts a list of desired filenames and content types.
2. The service uses the AWS SDK (configured for MinIO locally) to generate a unique, time-limited (e.g., 15 minutes), pre-signed `PUT` URL for each requested file.
3. The endpoint returns a list of objects, each containing the unique generated object key and its corresponding pre-signed URL.
4. The endpoint is secured with JWT authentication.
5. An integration test verifies that a valid pre-signed URL is generated and can be used to execute an HTTP PUT request directly against the Testcontainers MinIO instance.

## Tasks / Subtasks
- [ ] Task 1: Add AWS SDK Dependencies (AC: 2)
    - [ ] Add `software.amazon.awssdk:s3` and `software.amazon.awssdk:s3-presigner` to `build.gradle`. (Use the AWS SDK v2).
- [ ] Task 2: Configure S3 Client (AC: 2)
    - [ ] Create `S3Config.java` in `com.example.birdlensapi.config`.
    - [ ] Create an `S3Presigner` bean pointing to the MinIO endpoint (`app.s3.endpoint`).
    - [ ] Configure the bean to use the credentials from `app.s3.access-key` and `app.s3.secret-key`, and explicitly set `forcePathStyle(true)` (which is required for MinIO).
- [ ] Task 3: Build the Storage Service (AC: 2)
    - [ ] Create `S3StorageService.java` in `com.example.birdlensapi.storage`.
    - [ ] Implement `generatePresignedUploadUrls(List<String> filenames)`.
    - [ ] Ensure filenames are sanitized and prepended with a unique identifier (e.g., `UUID.randomUUID() + "-" + filename`) to prevent overwriting.
- [ ] Task 4: Create Controller (AC: 1, 3, 4)
    - [ ] Create `PostController.java` mapped to `/api/v1/posts`.
    - [ ] Create a `PresignedUrlRequest` DTO containing the list of filenames/content types.
    - [ ] Create a `PresignedUrlResponse` DTO containing `objectKey` and `presignedUrl`.
    - [ ] Implement the `POST /media/request-upload` endpoint.
- [ ] Task 5: Integration Testing (AC: 5)
    - [ ] Update `AbstractIntegrationTest` to include a MinIO Testcontainer (using image `minio/minio:latest`).
    - [ ] Ensure the container runs the command `server /data` and creates the `birdlens-media` bucket on startup.
    - [ ] Create `PostCreationIntegrationTest` (or `S3IntegrationTest`) that requests a URL and successfully PUTs a dummy text file to it.

## Dev Notes
- **MinIO Path Style**: MinIO requires "path-style" access, unlike standard AWS S3 which uses "virtual-hosted-style" access. You *must* configure `s3Configuration(S3Configuration.builder().pathStyleAccessEnabled(true).build())` on your S3 client builder, or it will try to resolve `http://birdlens-media.localhost:9000` and fail.
- **Security Check**: This endpoint must be protected by the `JwtAuthFilter`. You do not need to check `@AuthenticationPrincipal` inside the controller unless you want to append the user's ID to the S3 object key (e.g., `users/{userId}/{uuid}-{filename}`). This is actually highly recommended for better object organization.

### Testing
- You can use Spring's `RestTemplate` or `WebClient` inside your integration test to perform the actual `PUT` request to the pre-signed URL returned by your API.
