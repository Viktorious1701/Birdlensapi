# Story 4.3: Create Post API & Event Publishing

## Status
Draft

## Story
**As a** Mobile Client,
**I want** to create a new "Sighting" post with text and media references, receiving an immediate successful response,
**so that** I am not blocked while media processing happens asynchronously.

## Acceptance Criteria
1. An endpoint `POST /api/v1/posts` accepts post metadata (content, location, etc.) and a list of object storage keys/URLs corresponding to the uploaded files.
2. The service creates a new record in the `posts` table.
3. The service creates records in the `post_media` table with a `processing_status` of `PENDING`.
4. The service publishes a `PostCreatedEvent` message to a RabbitMQ exchange (`posts.exchange`) with the `post_id`.
5. The endpoint immediately returns a `201 Created` with the new post's data to the client.
6. A Testcontainers-backed integration test verifies that the database records are created and a message is successfully published to RabbitMQ.

## Tasks / Subtasks
- [ ] Task 1: Configure RabbitMQ (AC: 4)
  - [ ] Add `spring-boot-starter-amqp` to `build.gradle`.
  - [ ] Create `RabbitMQConfig.java` in `com.example.birdlensapi.config`.
  - [ ] Declare a DirectExchange named `posts.exchange`.
  - [ ] Declare a Queue named `image-processing-queue`.
  - [ ] Create a Binding between the queue and the exchange using the routing key `post.created`.
  - [ ] Configure `Jackson2JsonMessageConverter` so RabbitMQ messages are serialized as JSON instead of byte arrays.
- [ ] Task 2: Create DTOs & Event Payload (AC: 1, 4, 5)
  - [ ] Create `CreatePostRequest` (content, lat, lng, locationName, privacyLevel, type, sightingDate, taggedSpeciesCode, and a List of `mediaKeys`).
  - [ ] Create `PostResponse` mapping the saved entity.
  - [ ] Create `PostCreatedEvent` record containing the `postId`.
- [ ] Task 3: Implement PostService (AC: 2, 3, 4)
  - [ ] Add `createPost(CreatePostRequest request, String userEmail)` to `PostService`.
  - [ ] Fetch the User and (optionally) validate the BirdTaxonomy if a `taggedSpeciesCode` is provided.
  - [ ] Map the `lat`/`lng` to a PostGIS `Point` using `GeometryFactory`.
  - [ ] Save the `Post` and iterate through `mediaKeys` to save `PostMedia` entities with `ProcessingStatus.PENDING`.
  - [ ] Inject `RabbitTemplate` and call `convertAndSend("posts.exchange", "post.created", new PostCreatedEvent(post.getId()))`.
- [ ] Task 4: Create Controller (AC: 1, 5)
  - [ ] Add `@PostMapping` to `PostController`.
  - [ ] Accept `@Valid @RequestBody CreatePostRequest`.
  - [ ] Return `201 Created` with `ApiResponse.success(postResponse)`.
- [ ] Task 5: Integration Testing (AC: 6)
  - [ ] Create `PostCreationIntegrationTest` extending `AbstractIntegrationTest`.
  - [ ] Verify the DB insertion.
  - [ ] Use a `RabbitAdmin` or intercept the `RabbitTemplate` to verify the message arrived on the `image-processing-queue`.

## Dev Notes
- **Architecture Reference**: Refer to the Request Flow in `docs/architecture/high-level-architecture.md`. The client creates the post *after* they finish uploading to the pre-signed URLs from Story 4.2.
- **Geospatial Mapping in Java**: To convert the DTO's `lat`/`lng` to a PostGIS `Point` for the entity, use JTS:
  ```java
  GeometryFactory geometryFactory = new GeometryFactory(new PrecisionModel(), 4326);
  Point locationPoint = geometryFactory.createPoint(new Coordinate(request.lng(), request.lat()));
  ```
  *(Remember: X is Longitude, Y is Latitude!)*
- **Transactionality**: Ensure `@Transactional` is on the `createPost` method so the database commit and the RabbitMQ publish happen as an atomic unit (or fail together).

### Testing
- `AbstractIntegrationTest` already configures the RabbitMQ Testcontainer. You may need to inject `RabbitTemplate` into your test to consume the message directly from the queue to assert it was published correctly.
